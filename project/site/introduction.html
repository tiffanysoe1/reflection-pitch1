Introduction & Basics  

02.1 BASICS

What is Open-source LLMs?

Open Source models are AI models whose parameters are available to download. There are three taxonomies of openness:

  • Fully closed — API access only (OpenAI GPT-4, Anthropic Claude)
  • Weights-open — model weights released, training data closed (Meta LLaMA, Mistral, DeepSeek)
  • Fully open — weights + data + training process (OLMo, some Hugging Face models)

What is Sovereign AI?

Leaders are concerned about their ability to run AI systems with some level of independence from foreign technology providers. Sovereign AI is an entire ecosystem — owning the physical layer, keeping the country's data within its borders and governance.

Instead of waiting for private companies to do anything, the government acts as the primary customer. They identify the biggest needs — healthcare, banking — and pledge to use and fund AI for those specific areas. The government sets up rules on who is allowed to see the data, how it's encrypted, and what happens if the system is hacked.

The four pillars of AI sovereignty:

  • Compute — domestic access to GPU infrastructure and energy
  • Data — national data assets, localization, and governance frameworks
  • M — ability to train, fine-tune, or at minimum adapt frontier models
  • Governance — regulatory control, auditability, and value alignment

The five strategic advantages of Open-Source LLMs

1. Infrastructure Independence

  • Open weights can be deployed on domestic hardware — no API calls to foreign servers
  • Eliminates dependency on foreign cloud providers for critical government workloads
  • Allows 'air-gapped' deployment for defence, intelligence, and sensitive data applications

2. Cost Control at Scale

  • Enterprise and government users can run open models on their own infrastructure, controlling GPU costs directly
  • Avoids per-token pricing from closed providers, which becomes prohibitive at national scale
  • Reflection AI (Misha): 'Once you get into large enterprise territory, by default you want an open model. You can run it on your infrastructure. You can control its costs.'

3. Customisation and Localisation

  • Open weights enable fine-tuning on domestic language, law, culture, and sensitive datasets
  • Cultural alignment: a model trained on local data reflects local values, reducing the risk of foreign value imposition

4. Auditability

  • Open weights allow independent security audits — critical for government and defence use cases
02.2 REFLECTION AI'S BUSINESS

Where Reflection is focusing today

Context Reflection shifted towards open models after realizing there wasn't a strong frontier base model available for RL training. Ioannis mentioned his focus is now building that base model. Applications come later.
  • Reflection aims to build RL at scale — using RL during the pre-training process to help models learn "trajectories" rather than just predicting the next word.
  • The main challenge is that scaling up RL is even more challenging than scaling up LLMs. The machine learning systems stack is challenging. RL at scale also suffers from reward hacking and many other problems.
  • One real-world impact is to create an AI system that navigates the entire software development cycle, not just code generation. While LLMs are excellent at writing isolated code, end-to-end workflows have many more parts. How do you scale up your system to generalise across all domains?

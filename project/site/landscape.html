04 / 08 Client & competitor landscape

Client & Competitor Landscape

A working map of enterprises & sovereign AI—lessons learned from Timothee's (Mistral) detailed interview.

04.1 — WHAT IS IMPORTANT?
SLICE A

Introduction

According to a recent interview with Timothee(Mistral), the real value in enterprise is yet to be unlocked. There is a need for the rest of the stack: the serving platform, tooling and expertise to get into enterprise workflows to help that information.

This similar to what Misha mentioned in Nvidia GTC

SLICE B

Pre-training/Fine-Tuning Service

Some enterprise data is proprietary or domain-specific. When internal data is so new, and your model is trained on the public web, you need alot of tokens to understand it. In this case, customized pre-training is recommended. A company chose fine-tuning for efficiency reasons - smaller models that achieve strong performance on a narrow task.

SLICE C

Coding Agents

According to Timothee, Enterprise do not really think agents are useful. Complex workflows are preferred. The issue of agents is autonomy - this agent has access to some data that is privileged, but this other agent is publishing it to something that is public.You might have some governance concerns where some agent is acting on something very critical, and you don't know that the data it got has been approved.

04.2 — WHAT IS IMPORTANT?

Mistral

My interpretation is that Mistral is less ambitious about being the best "open-model" because they saw the difficulty of translating intelligence into ROI due to tooling, software stacks, etc.

A
Legacy Codebases

Migrating a massive codebase is a high risk, stressful operation that CTOs generally want to avoid. Mistral stack can be deployed on premise or within the customer's own virtual private cloud. This sounds similar to the problem Asimov was trying to solve.

B
Data

Companies are worried of data leaking through departments. They are currently building AI projects that are siloed and full value is only released when the company is confident enough to break the siloes and let AI access the entire context. As soon as you have agents running in the background, set them to run a bunch of ETL, demand for tokens will jump. For now, the boring software work must be done.

C
The ambitious product vision

Instead of Mistral having to manually help every customer, the goal is to provide a self-serve system. Customers should be able to easily gather their private, messy IP and "consolidate" it themselves. The platform should allow an enterprise to build a bespoke model that understands engineering. Timothee acknowledged this is far from possible.

Caveat I am drawing this map from public posts, podcasts, and a couple of conversations with friends inside the field. The internal version of this map you have is almost certainly more accurate. If there’s a slice I’m missing, I’d like to be told.
